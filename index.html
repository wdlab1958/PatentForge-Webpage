왜 PatentForge 인가?

PatentForge는 변리사를 대체하는 시스템이 아닙니다. 변리사가 가장 잘하는 일 — 법적 판단 — 에 시간을 쓰도록, 반복적이고 형식적인 작업(IDF 정형화·선행기술 검색·청구항 단일문장 검증·단락 번호 부여·도면 참조부호 일관성)을 자동화하는 변리사 협업 도구(Tooling)입니다. 변리사법 §2 저촉을 회피하기 위해 모든 산출물은 변리사 명의로 출원되며, 시스템은 측정 가능한 metric만 보고합니다.

⌘

변리사 협업 도구 (Tooling)

대리 행위는 변리사 업무. PatentForge는 도구만 제공합니다. 운영 환경에서 ENFORCE_ATTORNEY_REVIEW=true 설정 시 reviewer/admin 토큰만 최종 승인 가능.

⊠

무인 출원 절대 불허

HUMAN_REVIEW(Stage 10) 게이트는 변리사법 §2에 따라 100% 강제. 어떤 자동 통과 조건도 존재하지 않습니다. 변리사가 ✓ 승인 / ↺ 수정 / ✗ 포기 중 직접 결정.

◇

측정 가능한 거절 사유

"AI가 거부했습니다" ❌ → "독립항 단어 수 87 미만, KIPO-CLAIM-WORDS 검증 실패" ✓. 모든 거절은 정규식·AST·LLM 검증의 명시적 metric으로 설명됩니다.

▣

Hash-Chained Audit

모든 변리사 결정·LLM 호출·비용을 PostgreSQL audit_trail에 SHA-256 hash chain으로 5년간 영구 보존. 향후 무효심판·책임 추궁 시 증거 자료.

◐

비용 회로 차단기

사건당 $20 한도. 초과 시 CostCircuitBreaker가 즉시 파이프라인 정지. 평균은 $4–12, 최대 비용은 Claims 단계($2–5)와 Spec 단계($1–3)에 집중.

◈

오케스트라 모델

LangGraph Supervisor가 8명의 Specialist Agent를 지휘. IntakeAgent → PriorArtAgent → NoveltyAgent → ... 각 에이전트는 자기 도구만 사용 가능. 권한 분리가 코드 레벨에서 강제됩니다.

Design Principles

절대 위반하지 않는 6대 설계 원칙

한국 특허법(§29 신규성·진보성, §42 명세서 기재요건), 변리사법 §2, KIPO 별지 제15·16호 양식의 요건에서 도출된 6가지 원칙. 이 원칙은 어떤 경우에도 우회하지 않습니다.

PRINCIPLE 01

변리사법 §2 저촉 방지

특허에 관한 사항의 대리는 변리사 업무. 시스템은 도구만 제공하고, 출원은 변리사 명의로 진행. PatentForge가 변리사를 대체한다고 표현하지 않습니다.

PRINCIPLE 02

HITL 게이트 100% 강제

Novelty 회색지대(0.5~0.7) 게이트와 Stage 10 Review 게이트는 변리사 결정 없이 절대 통과 불가. 자동 승인 조건은 존재하지 않습니다.

PRINCIPLE 03

측정 가능한 metric만 출력

모든 거절 사유는 정규식 매치·AST 분석·LLM 분류기의 결과로 설명됩니다. "AI가 그렇게 판단했습니다" 형태의 불투명 응답을 산출하지 않습니다.

PRINCIPLE 04

BLOCKER 위반은 직진 차단

QAComplianceAgent의 BLOCKER 1개라도 발견되면 다음 단계로 진행 불가. 자동 회귀 또는 변리사 수정 필수. 출원 자체가 거부되는 사항을 묵인하지 않습니다.

PRINCIPLE 05

영업비밀 마스킹 의무

IDF의 confidential_terms에 입력된 용어는 외부 LLM 호출 시 자동 마스킹. ZDR(Zero Data Retention) 계약된 모델만 영업비밀 처리에 사용 허용.

PRINCIPLE 06

5년 영구 audit log

변리사 검토 사실·LLM 호출·비용은 SHA-256 hash chain으로 5년간 영구 보존. 향후 등록 거절 또는 무효심판 시 책임 추적이 가능하도록 설계되어 있습니다.

Safety Net

잘못된 출원을 막는 5대 안전 장치

파이프라인 중 5개 지점에 안전 장치를 배치하여, 발명자의 자기 진술의 과신·LLM의 환각·청구항의 회피 가능성·KIPO 양식 위반·법적 책임 분리 부재로 인한 사고를 사전 차단합니다.

GATE 1 · STAGE 3

진보성 회색지대 HITL

NoveltyAgent 점수 0.5~0.7 회색지대 진입 시 자동 진행 금지. 변리사 코멘트 + 진행/보강/포기 결정 강제.

hitl_novelty :: blocking
GATE 2 · STAGE 5

청구범위 강건성 게이트

AutoGen Red Team의 Robustness 점수 0.85 미달 시 자동 재작성. 3회 실패 시 변리사 직접 작성 게이트로 회귀.

robustness ≥ 0.85
GATE 3 · STAGE 9

KIPO 양식 BLOCKER 차단

QAComplianceAgent의 40+ 검증 규칙 중 BLOCKER 1개라도 발견되면 다음 단계 진행 불가. 자동 수정 시도 후 변리사 검토.

qa.blocker == 0
GATE 4 · STAGE 10

변리사 최종 검토 강제

무인 출원 절대 불허. 변리사가 ✓ 승인 / ↺ 수정 / ✗ 포기 중 직접 결정. 코멘트는 청구항·단락 단위 anchor로 구조화 입력.

attorney.decision != null
GATE 5 · STAGE 12

Audit Log 영구 보존

변리사 결정·에이전트 호출·비용을 5년간 영구 보존. SHA-256 hash chain으로 사후 변조 불가. 무효심판·책임 추궁 시 증거.

retention 5y · sha256-chained
12-Stage Pipeline

발명 → 출원, 12단계 자동 워크플로우

자유 서술의 발명 신고서를 받아 KIPO 출원 가능 .docx까지 변환하는 12단계 직렬 파이프라인. Intake → PriorArt → Novelty → TechImpl → Claims → Spec → Abstract → Figures → QA → Review → Assembly → Done. HITL 게이트는 Novelty 직후·Review 단계 두 곳에 설치되며, 그 외 단계는 점수 임계값 미달 시 자동 회귀하는 루프 구조입니다.

Intake

발명 정보 수집

자유 서술 → 17필드 IDF. Socratic 5–8 라운드 대화로 누락·모호 필드 보강.

Tier 3 · Qwen3-8B
02

PriorArt

선행기술 조사

03

⚠ HITL · Tier 1 · Sonnet 4.6
04

기술 구현 방법론

08

변리사 최종 검토

Compliance Tiers

위반의 등급, BLOCKER · MAJOR · MINOR · INFO

QAComplianceAgent(Stage 9)는 40+ 검증 규칙의 결과를 4단계 심각도로 분류합니다. BLOCKER는 출원 자체가 거부되는 사항, MAJOR는 거절이유 통지 가능성이 높은 사항, MINOR는 자동 수정 가능, INFO는 통계 항목입니다.

⛔

BLOCKER

출원 자체가 거부되는 사항. 직진 차단. 직전 단계로 자동 회귀 또는 변리사 수정 필수.

예: 단일 문장 위반, 단락번호 누락, 청구항 인용 순환
⚠

MAJOR

거절이유 통지 가능성이 높은 사항. 변리사 코멘트 다수 예상. 5개 이상 시 HITL 권고.

예: 도면 참조 불일치, 청구항 용어 명세서 미정의
◇

MINOR

자동 수정 가능 또는 권고 수준. 출원에는 지장 없음. 자동 통과 + 보고서에 기록.

예: 폰트, 여백, 단락 번호 형식 자잘한 차이
ⓘ

INFO

정보 제공용 통계 항목. 검토 부담 없음. 변리사 참고용.

예: 단어 수 분포, 단락 길이 분포
Statutory Mapping

한국 특허법 · 변리사법과의 1:1 매핑

PatentForge의 자동 검증 규칙과 HITL 게이트는 한국 특허법 §29(신규성·진보성)·§42(명세서 기재요건)·변리사법 §2(자격 보장)에서 직접 도출되었습니다. 시스템 결정은 항상 명시된 조항으로 traceable 합니다.

특허법 §29 — 신규성·진보성 PatentForge 대응

  • §29①출원 전 공지된 발명과 동일하지 않을 것 → NoveltyAgent 단일/결합 인용 위협도 별도 평가
  • §29②통상의 기술자가 용이하게 도출할 수 없을 것 → 3-페르소나 합의 + 회색지대 0.5~0.7 HITL
  • POSITA해당 분야 평균적 지식 + 5년 실무 → 시스템 프롬프트에 수준 명시
  • 예상 효과"통상의 기술자가 예상하지 못한 효과" 발굴 → IDF 효과 vs 선행기술 효과 정량 비교

특허법 §42 — 명세서 기재요건 PatentForge 대응

  • §42③1호실시 가능 명세서 → TechImplAgent 1차 실시예 + 대안 2~5 + 의사코드

변리사법 §2 — 변리사 자격 보장 법적 분리

  • 대리 행위특허에 관한 사항의 대리는 변리사 업무. PatentForge는 대리 행위 금지, 도구만 제공.
  • HITL Gate 4Stage 10 Review = 100% 인간. 자동 통과 조건 없음. 변리사 결정 없이 진행 불가.
  • 운영 강제운영 환경 ENFORCE_ATTORNEY_REVIEW=true → reviewer/admin 토큰만 최종 승인.
  • 비대면 검토화상회의·이메일도 변리사법상 유효. 단, audit log + 변리사 전자서명 필요.
  • 명의 출원모든 산출물은 변리사 명의로 출원. PatentForge는 KIPOnet 출원 자체를 수행하지 않음.

'다운로드' 카드에서 원하는 형식 선택. fetch+blob + JWT 인증으로 안전 전송. 링크 TTL은 24시간이며 재생성 가능.

Security

영업비밀과 audit log를 암호학적으로 보호

발명은 출원 전까지 영업비밀입니다. PatentForge는 외부 LLM 호출 시 자동 마스킹, ZDR(Zero Data Retention) 계약 모델만 사용, 모든 호출의 SHA-256 hash chain 보존으로 사후 변조를 암호학적으로 차단합니다.

인증 · 권한 RBAC

  • JWT 24h TTL모든 API 호출에 Bearer 토큰 필요. 만료 시 재로그인.
  • RBAC 3-Roleinventor / reviewer(변리사) / admin. 운영 환경에서 reviewer/admin만 최종 승인 가능.
  • fetch+blob 다운로드다운로드 요청에도 인증 헤더 자동 첨부. 다이렉트 URL 차단.
  • Squid 단일 IP외부 API(KIPRIS/USPTO 등)는 단일 프록시 경유. quota 관리 + 감사 추적.

영업비밀 · 데이터 보호 Zero Data Retention

  • confidential_termsIDF 입력 시 발명자가 직접 표시한 용어는 외부 LLM 호출 시 자동 마스킹.
  • ZDR 모델만 사용Anthropic 엔터프라이즈 ZDR 계약 모델만 confidential 처리에 사용. 학습·검토 보존 없음.
  • Redis 24h 캐시PriorArt 검색 결과 캐싱. 발명 본문은 캐시 키에 들어가지 않음.
  • 5년 보존 → 삭제사건 종료 후 5년 보존(특허 심사 대응). 그 이후 자동 삭제. 사용자 요청 시 즉시 삭제(right to erasure).
  • SHA-256 hash chain변리사 결정·LLM 호출·비용은 hash chain으로 영구 보존. 사후 변조 시 chain 깨짐.

단기는 KIPO 출원 안정성, 중기는 거절이유통지 후속 워크플로우(의견서·보정서) 자동화, 장기는 PCT·USPTO·EPO 다국 출원 지원. 궁극적으로는 변리사가 법적 판단에만 시간을 쓰도록 하는 표준 협업 도구가 되는 것이 목표입니다.

의도적으로 *하지 않을* 일

❌ 변리사 대체 (변리사법 §2 저촉)
❌ 무인 출원 (HITL 게이트 우회 불가)
❌ 발명 본문을 외부 학습에 제공
❌ KIPOnet 시스템 자동 출원 (책임 분리)

PatentForge는 후자에 집중합니다. 형식 검증·BLOCKER 차단·hash-chained audit으로 변리사가 안심하고 명의를 빌려줄 수 있는 도구가 되는 것이 우리의 목표입니다.
