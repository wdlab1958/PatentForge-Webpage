왜 PatentForge 인가?

PatentForge는 변리사를 대체하는 시스템이 아닙니다. 개인 발명가의 진입 장벽을 200만원 → 5만원으로 낮추고, 기업 R&D 특허 담당자의 영업비밀을 사내에 가두며, AI 우호 변리사의 형식 작업을 자동화하는 도구(Tooling)입니다. 변리사법 §2 "대리"를 피하기 위해 시스템은 사용자 명의로 출원하지 않고, 법률 효과를 발생시키지 않으며, 위임 계약을 체결하지 않습니다. 변리사법 §21 단서("본인이 직접 행하는 경우")에 따라 발명자가 자기 책임 하에 셀프 출원하는 Mode B도 지원합니다.

⌘

도구 (Tooling), 대리 아님

PatentForge는 산출물(.docx)만 사용자에게 전달하며, KIPO 출원은 사용자 본인 또는 변리사가 수행합니다. SaaS 이용약관만 체결, 위임 계약 없음.

⊠

PatentForge가 만드는 3 명의 사용자

PatentForge는 모든 발명자를 위한 도구가 아닙니다. 다음 3가지 명확한 페르소나의 일을 가장 잘 돕도록 설계되었으며, 각 페르소나는 다른 운영 모드를 선택하고, 다른 산출물을 사용하며, 다른 책임 경계를 가집니다. 나머지 사용자(예: 핵심 IP 대형 사건의 사내 법무팀, 분쟁 중인 특허의 무효심판 대응)는 직접 변리사 사무소를 권장합니다.

이 외 사용자는 누구를 위한 도구가 아닌가?

대형 IP 분쟁 중인 발명 · 핵심 권리범위가 사업 전략을 좌우하는 사건 · 무효심판 대응이 필요한 사건 은 PatentForge 단독 사용을 권장하지 않습니다. 이런 사건은 도구로 가속할 수 있는 영역이 아닌, 변리사 사무소의 법적 전문성과 누적 경험이 결정적인 영역입니다. PatentForge는 이 영역의 변리사 보조 도구로서만 의미가 있으며, 자동 출원이나 셀프 출원은 권장하지 않습니다.

Legal Positioning · 변리사법 §21 단서

"본인이 직접 행하는 경우" — 단서가 셀프 출원을 허용

한국 변리사법 §21은 변리사 아닌 자의 대리 행위를 금지하지만, 본문 단서가 본인 직접 행위는 허용합니다. PatentForge는 이 단서에 정확히 맞춰 설계되었습니다. PatentForge가 "대리"가 아닌 "도구"로 분류되려면 다음 3 조건을 모두 피해야 합니다 — ① 타인 명의 행위 금지 ② 법률 효과 발생 금지 ③ 위임 계약 체결 금지.

변리사법 §2 — 변리사 업무 법문

  • 본문변리사는 특허청 또는 법원에 대하여 특허·실용신안·디자인·상표에 관한 사항을 대리하고 그 사항에 관한 감정과 그 밖의 사무를 수행하는 것을 업으로 한다.
  • "대리"① 타인의 명의로 ② 법률 효과 발생 ③ 타인의 위임을 받음 — 3 요소 모두 충족 시 변리사 독점 업무
  • 처벌§54 — 5년 이하 징역 또는 5천만원 이하 벌금 + 민사 (금지 청구·손해배상) + 행정 (사업 중지)

변리사법 §21 — 업무영역 + 단서 PatentForge 근거

  • 본문변리사가 아닌 자는 §2 업무를 영위하지 못한다.
  • 단서 ①다른 법률에 특별한 규정이 있는 경우
  • 단서 ②본인이 직접 행하는 경우 — Mode B의 법적 근거
  • 적용 범위발명자 본인 (개인 명의) ✅ · 회사 대표이사 (법인 명의) ✅ · 회사 대표가 위임한 직원 (법인 명의) ✅

✓합법 — PatentForge 운영자가 안전한 행위

  • 발명자 본인이 본인 발명을 본인 명의로 출원 (§21 단서)
  • 법인 대표이사가 자기 회사 발명을 법인 명의로 출원
  • 회사 대표가 위임한 직원이 법인 명의로 출원 (위임장 첨부)
  • AI 도구가 사용자 발명에 대한 명세서 초안 생성 (도구 제공)
  • 양식 검증 결과 표시 (BLOCKER / MAJOR / MINOR)
  • 자체 자동 검증 점수 (진보성·실시가능성·회피강건성) 제공
  • 변리사 사무소 연결 (제휴 — 알선료 0원)

✗위법 — 처벌 위험 행위

  • 비변리사 컨설턴트가 타인 발명을 수임 대리하여 출원
  • AI 도구가 사용자 명의로 KIPO에 자동 출원 ("대리" 해당)
  • "변리사를 대신하여 출원해드립니다" 광고
  • 사용자 발명에 대한 단정적 법적 의견 ("이 발명은 진보성이 있다")
  • "AI 변리사" / "변리사 대체 서비스" 표현 사용
  • 변리사 사무소 알선·중개 수수료 수수
  • "특허 등록 보장" / "100% 등록 가능" 표현
⚠

회색 지대 — 자동 출원 기능 (현 시점 미구현)

만약 PatentForge가 KIPOnet API로 사용자 명의 자동 출원을 추가한다면 §2 "대리" 위반 가능성이 높습니다. 본 가이드는 MVP v1.0에서 자동 출원 기능을 구현하지 않습니다. Phase 3 이후 변리사회 정식 자문 후 도입 검토. 자동 출원이 필요한 경우의 방어 장치는 ① 사용자 명시적 클릭 + 본인 인증 또는 ② 자동 출원을 제휴 변리사 사무소에 위임의 두 가지뿐.

Design Principles

절대 위반하지 않는 6대 설계 원칙

한국 특허법(§29 신규성·진보성, §42 명세서 기재요건), 변리사법 §2, KIPO 별지 제15·16호 양식의 요건에서 도출된 6가지 원칙. 이 원칙은 어떤 경우에도 우회하지 않습니다.

PRINCIPLE 01

변리사법 §2 저촉 방지

특허에 관한 사항의 대리는 변리사 업무. 시스템은 도구만 제공하고, 출원은 변리사 명의로 진행. PatentForge가 변리사를 대체한다고 표현하지 않습니다.

PRINCIPLE 02

모드별 게이트 코드-레벨 강제

Mode A는 변리사 검토 100% 강제, Mode B는 면책 동의서 + 본인 인증 강제, Mode C는 사내 변리사 강제. 자동 승인 조건 없음 — 모드 우회 불가.

PRINCIPLE 03

측정 가능한 metric만 출력

모든 거절 사유는 정규식 매치·AST 분석·LLM 분류기의 결과로 설명됩니다. "AI가 그렇게 판단했습니다" 형태의 불투명 응답을 산출하지 않습니다.

PRINCIPLE 04

BLOCKER 위반은 직진 차단

QAComplianceAgent의 BLOCKER 1개라도 발견되면 다음 단계로 진행 불가. 자동 회귀 또는 변리사 수정 필수. 출원 자체가 거부되는 사항을 묵인하지 않습니다.

PRINCIPLE 05

영업비밀 마스킹 의무

IDF의 confidential_terms에 입력된 용어는 외부 LLM 호출 시 자동 마스킹. ZDR(Zero Data Retention) 계약된 모델만 영업비밀 처리에 사용 허용.

PRINCIPLE 06

5년 영구 audit log

변리사 검토 사실·LLM 호출·비용은 SHA-256 hash chain으로 5년간 영구 보존. 향후 등록 거절 또는 무효심판 시 책임 추적이 가능하도록 설계되어 있습니다.

Safety Net

잘못된 출원을 막는 5대 안전 장치

파이프라인 중 5개 지점에 안전 장치를 배치하여, 발명자의 자기 진술의 과신·LLM의 환각·청구항의 회피 가능성·KIPO 양식 위반·법적 책임 분리 부재로 인한 사고를 사전 차단합니다.

Statutory Mapping

특허법 §29 · §42와의 1:1 매핑

PatentForge의 자동 검증 규칙과 HITL 게이트는 한국 특허법 §29(신규성·진보성)·§42(명세서 기재요건)·변리사법 §2(자격 보장)에서 직접 도출되었습니다. 시스템 결정은 항상 명시된 조항으로 traceable 합니다.

특허법 §29 — 신규성·진보성 PatentForge 대응

  • §29①출원 전 공지된 발명과 동일하지 않을 것 → NoveltyAgent 단일/결합 인용 위협도 별도 평가
  • §29②통상의 기술자가 용이하게 도출할 수 없을 것 → 3-페르소나 합의 + 회색지대 0.5~0.7 HITL
  • POSITA해당 분야 평균적 지식 + 5년 실무 → 시스템 프롬프트에 수준 명시
  • 예상 효과"통상의 기술자가 예상하지 못한 효과" 발굴 → IDF 효과 vs 선행기술 효과 정량 비교

특허법 §42 — 명세서 기재요건 PatentForge 대응

  • §42③1호실시 가능 명세서 → TechImplAgent 1차 실시예 + 대안 2~5 + 의사코드
  • §42③2호발명의 효과 정량 기재 → 효과 계층(주1+부3~5) 강제, 정량 수치+비교 기준
  • §42④1호청구범위 명확성 → 단일 문장(마침표 1개) + 불명료 표현 정규식 차단
  • §42④2호명세서 뒷받침 → 청구항 용어를 명세서 본문에서 모두 정의했는지 자체 검증

※ 변리사법 §2 / §21 단서의 자세한 해설은 위쪽 "Legal Positioning" 섹션을 참고하십시오.

ResponseAgent의 모든 산출물에는 다음 면책 조항이 강제 첨부됩니다 — "본 초안은 일반 정보 제공 목적이며, 출원인 본인이 검토 후 수정·보완하거나 변리사에게 자문을 받으십시오. 의견제출 마감일(보통 2개월) 엄수. 본 초안 사용으로 인한 등록 거절·무효 등의 결과에 대해 PatentForge 운영자는 일체의 책임을 지지 않습니다." "이렇게 응답하면 됩니다" 같은 단정적 자문은 절대 출력하지 않습니다.

Roadmap · Vision

PatentForge의 최종 목표

단기는 KIPO 출원 안정성, 중기는 거절이유통지 후속 워크플로우(의견서·보정서) 자동화, 장기는 PCT·USPTO·EPO 다국 출원 지원. 궁극적으로는 변리사가 법적 판단에만 시간을 쓰도록 하는 표준 협업 도구가 되는 것이 목표입니다.

👤

개인 발명가에게 약속

• 비용 95~98% 절감 — 200만원 → 5만원
• KIPOnet 셀프 출원 가이드 6단계 PDF 자동 동봉
• 면책 동의서 + SHA-256 — 법적 보호
• 의견제출 대응 ResponseAgent — 거절이유통지서 OCR + 의견서 초안
• 13,000~20,000명 잠재 사용자

🏢

기업 R&D 담당자에게 약속

• 외부 LLM 호출 0건 — vLLM 로컬 + Squid 프록시 강제
• 영업비밀 외부 송신 0건 audit 자동 검증
• 4-관할 일괄 출원 (KIPO+USPTO+EPO+PCT)
• 법인 명의 강제 + 사내 변리사 등록
• 약 2,500개사 잠재 고객

AI 우호 변리사에게 약속
